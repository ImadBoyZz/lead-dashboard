# Lead Dashboard — Design Context

## Users

**Primary user**: Imad Bardid (Averis Solutions), solo internal operator. Opens the dashboard op desktop in ochtendreviewsessies (`/review` queue + status-check) en incidenteel door de dag heen bij alerts.

**Context**: Dit is een intern operations-tool voor cold outbound automation — geen klantgerichte UI, geen public page. Wordt gebruikt naast Telegram (voor alerts) en direct Vercel/n8n voor diepere debugging.

**Jobs-to-be-done**:
- "Ik open het dashboard 's ochtends en weet binnen 10 seconden of het systeem gisteravond en vannacht foutloos heeft gedraaid."
- "Als er iets mis gaat, pauzeer ik send met één klik en lees ik de laatste errors."
- "Ik zie of mijn warmup op schema loopt en of ik nog binnen budget zit."

## Brand Personality

3 woorden: **rustig, precies, operational**

Niet: flashy, consumer, gamified. Wel: observabililty-tool zoals Linear, Grafana, Stripe dashboard, Vercel. De UI is tot nu toe een lichte functionele app met dark-navy sidebar. Matching met Vercel/Stripe-patroon waar hij al bekend mee is.

## Aesthetic Direction

**Theme**: Light content area + donkerblauw-navy sidebar. Bestaande UI aanhouden — `/autonomy` wordt een extensie ervan, geen stijlbreuk.

**Referentie-ankers**:
- Linear issue-feed: compact, status-dots, uppercase-tracked sectielabels, subtle 1px row-borders
- Stripe dashboard: tabular-nums mono cijfers, grote numerieke waardes naast kleine context-text
- Vercel observability: status-dot met 2px glow, monospace numerics, restraint in kleur

**Anti-references**:
- Generic admin dashboards met icon-grid + identieke cards
- AI-templated "hero metric met purple-to-blue gradient"
- Glassmorphism, bounce-animaties, decoratieve sparklines
- Side-stripe borders (CRITICAL ban — never)

## Design Principles

1. **Numerics are the protagonist**: Elke metric is een grote mono-gezette waarde (tabular-nums) met kleine context-label erboven. Uppercase-tracked label, niet titel-gezet.
2. **Status-dots doen het zware werk**: Health van een sectie communiceert primair via kleur-dot (success/warning/danger/idle) met 2px glow, niet via banner-cards of emoji.
3. **Subtle over bold borders**: Sectie-scheiding via 1px borders in `rgba(15,23,42,0.06)`, niet via shadow-cards. Border-left >1px is verboden.
4. **Density over whitespace**: Dit is een ops-tool. Imad wil veel informatie in één viewport kunnen scannen — niet elk metric in een apart card met 48px padding.
5. **Nederlandstalig, direct, geen marketing-taal**: "Verstuurd vandaag 3/5", niet "Je hebt vandaag 3 van je 5 dagelijkse emails verzonden! 🚀".

## Typography

- Body/UI: Inter (bestaand, wijzig ik niet — out-of-scope herfont hele app)
- Numerics: **JetBrains Mono** (niet op reject-lijst, uitstekend voor tabular data). Toegepast via `--font-mono` token + `font-variant-numeric: tabular-nums`.
- Sectielabels: 11px uppercase, tracking 0.08em, muted kleur.

## Color System

Bestaande tokens (light mode, oklch-lookalike hex):
- `--color-background: #f8fafc` (zinc-50)
- `--color-foreground: #0f172a` (slate-900)
- `--color-card: #ffffff`
- `--color-card-border: #e2e8f0`
- `--color-muted: #64748b`
- `--color-accent: #2563eb` (blue-600)
- `--color-success: #16a34a` (green-600)
- `--color-warning: #d97706` (amber-600)
- `--color-danger: #dc2626` (red-600)
- Sidebar: `#1e293b` bg, `#e2e8f0` fg, `#2563eb` active

Nieuw voor /autonomy:
- `--color-surface-hover: #f1f5f9`
- `--color-border-subtle: rgba(15,23,42,0.06)`
- `--color-dot-glow-success/warning/danger` (2px glow halos op status-dots)
- `--font-mono` (JetBrains Mono fallback)

## Accessibility

- WCAG AA voor text contrast (bestaand meetbaar in gebruikte paren)
- `prefers-reduced-motion` respecteren voor polling-state transitions
- Keyboard navigation door feed-rows (expand via Enter/Space)
- Status-dots begeleid door text-label (kleur is nooit de enige indicator)
